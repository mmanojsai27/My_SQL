--TCL(TRANSACTION CONTROL LANGUAGE)
------------------------------------------
DEF:
----
* THE PROCESS OF COMBING ALL RELATED OPERATION INTO A SINGLE UNIT AND EXCUTING THEM ACCORDING TO THE RULE "EITHER ALL OPERATION SHOULD BE SUCCEED OR MORE SHOULD BE SUCCEED", IS CALLED TRANSCATION MANAGEMENT..

* A TRANSCATION I CONSIDERED AS A SINGLE UNIT OF WORK AND FOLLLOWS THE PRINCPLE:
--> "EITHER ALL OR NONE"

EXAMPLE: FUNDS TRANSFER..

CONSIDER A MONEY TRANSFER PROCESS BETWEEN TWO ACCOUNTS..
1.DEBIT AMOUNT FROM THE SENDER'S ACCOUNT.
2.CREDIT AMOUNT INTO THE RECEIVER'S ACCOUNT.

-->BOTH OPERATIONS SHOULD BE EXCUTED AS A SINGLE TRANSACTION..

>>IF THE AMOUNT IS DEBIT FROM THE SENDER'S ACCOUNT SUCCESSFULY BUT CREDITING THE RECOVER'S ACCOUNT FAILS. IT MAY LEAD TO THE DATA INCONSISTENCY PROBLEMS..

>THERE ARE TWO TYPES OF TRANSACTIONS SUCH AS:-
1) LOCAL TRANSACTION:-
-----------------------
IN LOCAL TRANSACTIONS, ALL OPERATIONS OF A TRANSACTION ARE EXECUTED IN THE SAME DATABASE..
EXAMPLE:-
--------
FUNDS TRANSFER FROM ONE ACCOUNT TO ANOTHER ACCOUNT WHERE BOTH ACCOUNTS BELONGS TO THE SAME BANK...

2)GLOBAL TRANSACTION:
---------------------
IN GLOBAL TRANSACTION, OPERATIONS OF A TRANSACTION ARE EXECUTED ON DIFFERENT DATABASES..
EXAMPLE:-
---------
FUNDS TRANSFER FROM ONE ACCOUNT TO ANOTHER ACCOUNT WHERE BOTH ACCOUNTS BELONGS TO THE DIFFERENT BANK...

ACID RULES:--
-------------
>ATOMICITY(A):-- ALL (OR) NOTHING..

>CONSISTENCY(C):-- THE DATABASE REMAINS IN A VAILD STATE

>ISOLATION(I):-- TRANSACTION DO NOT INTERAFCE WITH EACH OTHER

>DURABILITY(D):-- COMMITTED CHANGES ARE PERMANENT...

WHAT IS LOCKING:-
---------------
>>LOCKING IS A MECHANISM USED BY THE DATABASE TO CONTROL CONCURRENT ACCESS DATA..

>>IT ENSURES THAT MULTIPLE USERS ACCESS THE SAME DATA SIMULTANEOUSLY THE DATABASE MAINTAINS CONSISTENCY, INTEGRITY, AND PREVENT CONFLICTS..

1) SHARED LOCK
----------------
DEF:- A SHARED LOCK MEANS ALLOWS A TRANSACTION TO READ DATA.
MULTIPLE TRANSACTIONS CAN HOLD A SHARED LOCK ON THE SAME ROW..
NO TRANSACTION CAN MODIFY THE ROW UNTIL ALL SHARED LOCKS ARE RELEASED..

*CHARACTERISTICS:-
-------------------
READ ONLY 
MULTIPLE USERS CAN READ SIMULTANEOUSLY
BLOCKS UPDATES AND DELETES
-----------------------------------------------------------------------------------
2) EXCLUSIVE LOCK:-
DEF:- AN EXCLUSIVE LOCK ALLOWS A TRANSACTION TO MODIFY DATA..
WHILE AN EXCLUSIVE LOCK EXISTS..
NO OTHER TRANSACTIONS CAN READ USING A LOCKING READ OR MODIFY THAT ROW..
ONLY THE TRANSACTION HOLDING THE LOCK CAN UPDATE/DELETE THE DATA..

*CHARACTERISTICS:-
-------------------
USED FOR INSERT, UPDATE, DELETE.
ONLY ONE EXCLUSIVE LOCT AT A TIME.
BLOCKS BOTH READS(LOCKING READS) AND WRITES

3)INTENT LOCK
--------------
AN INTENT LOCK IS A TABLE-LEVEL LOCK THAT INDICATES A TRANSACTION INTENDS TO PLACE ROW-LEVEL.
IT HELPS MYSQL QUICKLY DETERMINE WHEATHER A TABLE-LEVEL LOCK CONFLICTS WITH EXISTING ROW LOCKS..
USERS DO NOT USUALLY CREATE INTENT LOCKS MANUALLY INNODB MANAGES THEM AUTOMATICALLY

REAL-TIME EXAMPLE
IMAGINE A LIBRARY..
SHARED LOCK -> MANY STUDENTS CAN READ THE SAME BOOK AT THE SAME TIME..

EXCLUSIVE LOCK --> ONE STUDENT BORROWS THE BOOK, SO NOBODY ELSE CAN READ OR BORROW IT UNTIL IT IS RETURNED.

INTENT LOCK --> THE LIBRARIN MARKS THAT SOMEONE IS ABOUT TO READ OR BORROW BOOKS FROM A SHELF, HELPING COORDINATE ACCESS WITHOUT CHECKING EVERY BOOK INDIVIDUALLY..
